# 工厂排单系统 - 销售功能改造说明

## 一、数据库改造 (models.py)

### 1. Product表新增字段
| 字段名 | 类型 | 说明 |
|--------|------|------|
| unit_price | Numeric(10,2) | 单价，单位元 |
| amount | Numeric(10,2) | 金额，自动计算=数量×单价 |
| unit | String(10) | 单位，默认"件" |

### 2. Order表新增字段
| 字段名 | 类型 | 说明 |
|--------|------|------|
| contact_person | String(50) | 联系人 |
| contact_phone | String(20) | 联系电话 |
| total_amount | Numeric(12,2) | 订单总金额 |
| payment_status | String(20) | 付款状态：unpaid/partial/paid |
| paid_amount | Numeric(12,2) | 已付金额 |

### 3. 新增Payment表
```python
class Payment(db.Model):
    id, order_id, amount, payment_date
    payment_method, remark, created_by, created_at
```

## 二、迁移脚本 (migrate_add_sales_fields.py)

- 使用方法: `python migrate_add_sales_fields.py`
- 回滚方法: `python migrate_add_sales_fields.py --rollback`

## 三、创建订单页面 (create_order.html)

### 新增功能
1. **客户信息区域**
   - 联系人输入框
   - 联系电话输入框

2. **产品明细区域**
   - 单价输入框（每个产品一行）
   - 金额自动计算并显示
   - 单位下拉选择

3. **颜色下拉固定选项**
   - 透明
   - 湖蓝
   - 草绿
   - 茶色
   - 灰色

4. **JavaScript金额计算**
   - 每行金额 = 数量 × 单价
   - 底部实时显示订单总金额
   - 处理空值和非法输入

## 四、后端路由 (routes/order.py)

### 修改要点
1. `create_order` 函数新增字段处理
2. 计算并保存 amount 和 total_amount

## 五、菜单结构调整 (dashboard.html)

### 新菜单结构
```
├─ 控制台（首页）          - 所有用户
├─ 报价管理               - 所有用户
├─ 创建订单               - 跟单员/管理员可见
├─ 订单列表               - 所有用户
├─ 生产任务               - 生产人员可见
├─ 付款记录               - 管理员可见
├─ 系统管理               - 管理员可见
│  ├─ 用户管理
│  └─ 全部订单
├─ 修改密码               - 所有用户
└─ 退出登录               - 所有用户
```

### 权限说明
| 角色 | 可用功能 |
|------|----------|
| admin | 全部功能 |
| admin_assistant | 报价管理、创建订单、订单列表 |
| clerks | 报价管理、创建订单、订单列表 |
| production | 控制台、订单列表、生产任务 |

## 六、订单状态扩展

| 状态值 | 显示名称 | 说明 |
|--------|----------|------|
| quoting | 报价中 | 新建订单默认状态 |
| confirmed | 已确认 | 客户确认后 |
| pending | 待生产 | 待安排生产 |
| producing | 进行中 | 生产中 |
| completed | 已完成 | 生产完成 |
| cancelled | 已取消 | 订单取消 |
| paused | 已暂停 | 生产暂停 |

---

## 七、使用注意事项

1. **金额精度**: 所有金额使用 `Numeric` 类型，避免浮点数精度问题
2. **迁移顺序**: 先运行迁移脚本，再更新代码
3. **数据兼容**: 迁移脚本兼容已有数据，新字段自动设置默认值
4. **权限控制**: 创建订单需要 `clerks` 及以上角色
